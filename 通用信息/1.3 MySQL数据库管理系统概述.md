1.3.1 什么是MySQL？

MySQL是最流行的开源数据库管理系统，由Oracle公司开发、发布和支持。

* MySQL是一个数据库管理系统

数据库是数据的结构化集合。它可以是任何东西，从一个简单的购物清单到一个图片库或公司网络中的大量信息。
要添加、访问和处理存储在计算机数据库中的数据，您需要一个数据库管理系统，如MySQL服务器。
由于计算机非常擅长处理大量数据，因此数据库管理系统在计算、作为独立实用程序或作为其他应用程序的一部分发挥着核心作用。

* MySQL数据库是关系型数据库

一个关系型数据库是将数据拆分到不同多张表中，而不是将所有数据放到一张表中。其数据结构会组织成物理文件以提高速度。具有数据库、
表、视图、行和列等对象的逻辑模型提供了灵活的编程环境。您可以在数据字段间设置不同的映射关系，如一对一、一对多、唯一性、必填性
或选填、或者不同表之间的"指针"。数据库通过这些规则，对于设计良好的数据库，应用程序永远不会看到不一致的、重复的、孤立的、过期
的或丢失的数据。

“MySQL”的SQL部分代表“结构化查询语言”。SQL是用于访问数据库的最常用的标准化语言。根据您的编程环境，您可以直接输入SQL(例如，生成报告)，
将SQL语句嵌入到用另一种语言编写的代码中，或者使用隐藏SQL语法的特定于语言的API。

SQL是由ANSI/ISO SQL标准定义的。自1986年以来，SQL标准一直在发展，已经有了几个版本。在本手册中，“SQL-92”指的是1992年发布的标准，
“SQL:1999”指的是1999年发布的标准，“SQL:2003”指的是标准的当前版本。我们使用短语“SQL标准”来表示任何时候的SQL标准的当前版本。

* MySQL是开源的

开源意味着任何人都可以使用或者修改MySQL。任何人可以从网上免费下载MySQL，您可以学习源代码然后根据您的需要去改写源代码。MySQL使用了GPL
[License](http://www.fsf.org/licenses)来定义在不同的情况下使用软件可以做什么，不可以做什么。当然，您可以从Oracle购买License去做您想做
的事。

* MySQL数据库服务器非常快速、可靠、可伸缩且易于使用。

如果这就是你想要的，你应该尝试一下。MySQL服务器可以轻松地运行在桌面或笔记本电脑上，以及您的其他应用程序、web服务器等，几乎不需要注意。
如果您将整个机器奉献给MySQL，您可以调整设置以利用所有可用的内存、CPU能力和I/O容量。MySQL还可以扩展到联网的机器集群。

MySQL服务器最初是为了比现有解决方案更快地处理大型数据库而开发的，并且已经成功地在高要求的生产环境中使用了几年。尽管在不断的开发中，
MySQL服务器今天提供了一组丰富而有用的功能。它的连通性、速度和安全性使MySQL服务器非常适合在Internet上访问数据库。

* MySQL服务器在客户端/服务器或嵌入式系统中工作。

MySQL数据库软件是一个客户端/服务器系统，它由一个多线程SQL服务器组成，支持不同的后端、几个不同的客户端程序和库、管理工具和各种应用程序编程接口(api)。


MySQL服务器还可以作为一个嵌入式多线程库，您可以将其链接到您的应用程序中，从而获得一个更小、更快、更易于管理的独立产品。


* 大量的贡献MySQL软件是可用的。

MySQL服务器有一组与用户密切合作开发的实用功能。您最喜欢的应用程序或语言很可能支持MySQL数据库服务器。

“MySQL”的官方发音是“My Ess Que Ell”(不是“My sequel”)，但是不介意你把它读成“My sequel”或者其他本地化的发音。


1.3.2 MySQL主要特性

本节介绍MySQL数据库软件的一些重要特性。在大多数情况下，路线图适用于MySQL的所有版本。有关在特定系列基础上引入MySQL的特性的信息，请参阅相应手册的“简要说明”部分:

* MySQL 8.0: [Section 1.4, “What Is New in MySQL 8.0”]()

* MySQL 5.7: [What Is New in MySQL 5.7]()

* MySQL 5.6: [What Is New in MySQL 5.6]()


#### 内部结构和可移植性

* 使用C/C++语音
* 使用各种不同的编译器进行测试
* 可以在各种平台运行, [参考](https://www.mysql.com/support/supportedplatforms/database.html.)
* 为了便于移植，在MySQL 5.5及以上版本中使用CMake。上一个系列使用GNU Automake、Autoconf和Libtool。
* 用Purify(一个商业内存泄漏检测器)和Valgrind(一个GPL工具)进行了[测试](http://developer.kde.org/~sewardj/)。
* 采用独立模块的多层服务器设计。
* 设计思想支持多线程，如果有可用的CPU，可以轻松使用多个CPU
* 支持事务、非事务存储引擎
* 使用带索引压缩的快速B-tree磁盘表（MyISAM）
* 设计了可以轻松添加存储引擎的接口，如果你想为内部数据库提供SQL接口，这将非常有用
* 使用了非常快的基于线程的内存分配系统
* 使用优化的nested-loop算法执行快速join
* 临时表采用了in-memory hash表
* 使用了大量优化的类库（尽可能快）实现SQL函数。通常在查询初始化之后根本没有内存分配
* 将服务器作为一个单独的程序提供给客户机/服务器网络环境中使用，并作为一个库提供给独立的应用程序。这些应用程序可以单独使用，也可以在没有可用网络的环境中使用。

#### 数据类型

* 非常多的数据类型：signed/unsigned integers 1,2,3,4,8 long, float,double,char, varchar, binary, varbinary, text, blob, date, time, datetime, timestamp, year, set, enum, OpenGIS spatial types。
参考[11章 数据类型]()

* 固定长度和可变长度的字符串类型

#### 语法和函数

* 查询的SELECT列表和WHERE子句中的完整操作符和函数支持。例如:

```
SELECT CONCAT(first_name, ' ', last_name) FROM citizen
WHERE income / dependentes > 10000 AND age > 30;
```

* 完全支持`GROUP BY`和`ORDER BY`子句。支持分组函数（`COUNT(), AVG(), STD(), SUM(), MAX(), MIN(), GROUP_CONCAT()`等等）

* 使用标准SQL和ODBC语法支持`LEFT OUTER JOIN` 和`RIGHT OUTER JOIN`

* 根据标准SQL语法，支持表和列上的别名。

* 支持删除、插入、替换和更新，以返回更改(受影响)的行数，或者通过在连接到服务器时设置标志来返回匹配的行数。

* 支持特定于mysql的`SHOW`语句，这些语句检索关于数据库、存储引擎、表和索引的信息。支持INFORMATION_SCHEMA数据库，根据标准SQL语法实现。

* 使用`EXPLAIN`语法显示优化器如何解析查询

* 函数名与表名或列名的独立性。例如，ABS是一个有效的列名。惟一的限制是，对于函数调用，函数名与其后面的“(”之间不允许有空格。见第9.3节[“关键词和保留词”]()。

* 您可以在同一条语句中引用来自不同数据库的表。

### 安全

* 非常灵活和安全的特权和密码系统，并支持基于主机的验证。

* 当您连接到服务器时，通过加密所有密码流量来实现密码安全性。

### 可伸缩性和限制

* 对大型数据库的支持。我们使用MySQL服务器，数据库包含5000万条记录。我们还知道一些用户使用的MySQL服务器有20万个表和大约50亿行。

* 每个表最多支持64个索引。每个索引可以由1到16列或列的一部分组成。InnoDB表的最大索引宽度是767字节或3072字节。见15.22节“[InnoDB限制]()”。MyISAM表的最大索引宽度是1000字节。参见16.2节“[MyISAM存储引擎]()”。对于CHAR、VARCHAR、BLOB或TEXT列类型，索引可以使用列的前缀。

### 连接

* 客户端可以使用如下几个协议连接到MySQL服务器:
    * 客户端可以在任何平台上使用TCP/IP协议
    * 在Windows系统上，如果服务器启动时启用了named_pipe系统变量，客户端可以使用命名管道进行连接。如果启动时启用了shared_memory系统变量，Windows服务器还支持共享内存连接。客户端可以通过使用`――protocol=memory`选项通过共享内存进行连接。
    * 在Unix系统上，客户端可以使用Unix域套接字文件进行连接。

* MySQL客户端程序可以用多种语言编写。用C编写的客户端库可用于用C或c++编写的客户端，也可使用任何提供C语言支持的客户端。

* 提供了用于C、c++、Eiffel、Java、Perl、PHP、Python、Ruby和Tcl的api，使得可以用多种语言编写MySQL客户端。参见第28章，[连接器和api]()。

* 连接器/ODBC (MyODBC)接口为使用ODBC(开放数据库连接)连接的客户机程序提供了MySQL支持。例如，您可以使用MS Access连接到MySQL服务器。客户端可以在Windows或Unix上运行。连接器/ODBC源可用。支持所有ODBC 2.5函数，以及许多其他函数。请参阅MySQL连接器/ODBC开发人员指南。

* 连接器/J接口为使用JDBC连接的Java客户机程序提供了MySQL支持。客户端可以在Windows或Unix上运行。连接器/J源可用。请参阅MySQL连接器/J [5.1开发人员指南]()。

* MySQL Connector/NET使开发人员能够轻松创建。NET应用程序，这些应用程序需要安全、高性能的MySQL数据连接。它实现了所需的ADO。NET接口并集成到ADO中。净意识的工具。开发人员可以使用他们选择的。net语言来构建应用程序。MySQL Connector/NET是一个完全托管的ADO。NET驱动程序写在100%纯c#。请参阅[MySQL连接器/NET开发人员指南]()。

### 本地化

* 服务器可以用多种语言向客户端提供错误消息。参见10.12节，“[设置错误消息语言]()”。

* 完全支持几种不同的字符集，包括latin1 (cp1252)、german、big5、ujis、几种Unicode字符集等等。例如，允许在表和列名中使用“?”, “?” and “?” 。

* 所有数据都保存在选定的字符集中。

* 排序和比较是根据默认字符集和排序规则进行的。当MySQL服务器启动时，可以更改这个(参见10.3.2节，“[服务器字符集和排序]()”)。要查看非常高级的排序示例，请查看`Czech`排序代码。MySQL服务器支持许多不同的字符集，可以在编译时和运行时指定。

* 服务器时区可以动态更改，各个客户机可以指定自己的时区。见5.1.14节“[MySQL服务器时区支持]()”。

### 客户和工具

* MySQL包括几个客户端和实用程序。这些程序包括命令行程序(如`mysqldump`和`mysqladmin`)和图形程序(如`MySQL Workbench`)。

* MySQL服务器内置了对SQL语句的支持，用于检查、优化和修复表。这些语句可以从命令行通过`mysqlcheck`客户端获得。MySQL还包括`myisamchk`，这是一个非常快速的命令行实用程序，用于在MyISAM表上执行这些操作。见第4章，[MySQL程序]()。

* MySQL程序可以执行` --help or -? `命令获得在线帮助。
